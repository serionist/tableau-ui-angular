Icons

This page demonstrates the use of Material Icons in the component library.

Getting started

Material Design Icons is an established icon set that is widely used in web development.
The component library provides a wrapper around these icons to make them easier to use.
There are two major ways that material icons can be added into your project:

  • Locally Hosted icons, imported via the Material Icons NPM Package:
    All the icons are hosted locally, no external web request is made
    Large size. Browser downloads ~4.5MB+ (uncompressed) to make icons available
  • Dynamic Icons, loaded from the Material Icons CDN:
    Only loads the used icons, small download size (a few kilobytes, depending on the number of icons used)
    No manual setup needed, used icons are automatically loaded the first time they are used
    (Optional) Icons loaded at least once are remembered and loaded automatically at every page load
    Hosted app needs access to Material Icons CDN, icons are not hosted locally
    Unloaded icons are blank until load from CDN completes

This sample app uses Dynamic Icons, but contains commented code to use Locally Hosted icons as well.

Usage

To add an icon to your app, use the <tab-icon> component and set the value attribute to the name of the icon you want to use.

This will render:
  • You can search for icons on Material Design Icons. Use the icon name as it appears in the Inserting the icon section on the right side panel after an icon is selected.
  • The size of the icon is 1em by default. Changing the font-size CSS property resizes the icon. No need to update width and height.
  • Icons can be colored using the color attribute. Possible values: primary, error, success. For any custom color, the color CSS property can be used.
  • Adding class="fill" will make the icon filled. Advanced attributes can be configured using the font-variation-settings CSS attribute.

Installing Dynamic Icons

Dynamic Icons works by automatically detecting which icons are used in tab-icon components and loads them from the Material Icons CDN.
This detection is done at runtime, so no manual setup is needed. Icons are loaded the first time they are used.
Optionally, icons that have been loaded at least once can be remembered and loaded automatically at every page load. This is done by saving the used icons into localStorage and is turned on by default.

  1. If you use modules, add the TableauUiIconModule to your module imports and configure it using forRoot:
    enableDynamicIcons: true turns the feature on. It is false by default
    enableDynamicIconsLocalStorageCache: false turns off saving used icons into localStorage. It is true by default.
